üîÑ 6. Pull Requests - Code Review Process

+

What is a Pull Request? ü§î

A Pull Request (PR) is a way to propose changes to a repository. It's like saying "Hey team, I've made some changes, please review and merge them!"

Pull Request Process

Developer pushes branch
Creates Pull Request
Team reviews code
Discusses changes
Approves & merges

Creating a Pull Request üìù

  1. Push your feature branch to GitHub
  2. Go to your repository on GitHub
  3. Click "Compare & pull request" button
  4. Write a clear title and description
  5. Select reviewers
  6. Click "Create pull request"
# Example PR workflow git checkout -b feature/user-profile # Make your changes... git add . git commit -m "Add user profile page with edit functionality" git push origin feature/user-profile # Then go to GitHub and create Pull Request

‚úçÔ∏è Writing Good Pull Requests

Title: Clear and descriptive
Description: What changes you made and why
Screenshots: For UI changes
Testing: How you tested your changes

üö® 7. Common Issues & Solutions

+

Merge Conflicts üí•

When two people edit the same line of code, Git doesn't know which version to keep.

# When you see a merge conflict git pull origin main # Auto-merging file.txt # CONFLICT (content): Merge conflict in file.txt # Open the file and you'll see: <<<<<<< HEAD Your changes ======= Their changes >>>>>>> branch-name # Edit the file to resolve conflict, then: git add . git commit -m "Resolve merge conflict"

Forgot to Create Branch üòÖ

# If you made changes on main branch by mistake git stash # Save your changes temporarily git checkout -b feature-branch # Create new branch git stash pop # Restore your changes git add . git commit -m "Move changes to feature branch"

Want to Undo Last Commit üîÑ

# Undo last commit but keep changes git reset --soft HEAD~1 # Undo last commit and discard changes git reset --hard HEAD~1

üõ°Ô∏è Golden Rules to Avoid Issues

‚Ä¢ Always git pull before starting work
‚Ä¢ Create feature branches for all changes
‚Ä¢ Commit frequently with clear messages
‚Ä¢ Test your code before pushing
